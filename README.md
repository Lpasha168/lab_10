# lab_myvector

Учебный проект: шаблон динамического массива `MyVector<T>` (header-only) и специализация `MyVector<bool>` с упаковкой элементов по битам в памяти типа `uint8_t`.

## Структура проекта

| Путь | Назначение |
|------|------------|
| `include/MyVector.h` | Библиотека в одном заголовке: основной шаблон и специализация для `bool` |
| `src/main.cpp` | Консольное демонстрационное приложение |
| `tests/vector_tests.cpp` | Юнит-тесты на Google Test |
| `CMakeLists.txt` | Сборка: цели `demo` и `run_tests`, подтягивание GTest через `FetchContent` |

## Требования

- **CMake** 3.16 или новее  
- **Компилятор C++17** (GCC, Clang, MSVC)  
- Для тестов при первой конфигурации нужен **доступ в интернет** (скачивание архива Google Test)

## Сборка

Из корня репозитория:

```bash
cmake -S . -B build
cmake --build build
```

### Одноконфигурационные генераторы (Ninja, Make)

Исполняемые файлы обычно лежат в `build/`:

- демо: `./build/demo`  
- тесты: `./build/run_tests`  
- через CTest: `cd build && ctest --output-on-failure`

### Мультиконфигурация (Visual Studio на Windows)

Укажите конфигурацию при сборке и при запуске CTest:

```powershell
cmake -S . -B build
cmake --build build --config Release
ctest --test-dir build -C Release --output-on-failure
```

Бинарники: `build/Release/demo.exe`, `build/Release/run_tests.exe` (путь может отличаться в зависимости от генератора и конфигурации).

### Цели CMake

- **`demo`** — пример использования `MyVector`  
- **`run_tests`** — исполняемый файл с тестами Google Test  

## Как устроена специализация `MyVector<bool>`

Обычный `MyVector<T>` хранит элементы подряд, как в массиве: один объект `T` на слот, адресуемый по индексу.

Для **`bool`** такой подход разорителен по памяти (как правило, под каждый логический флаг отводится целый байт или больше). В специализации данные хранятся **плотно**:

1. **Буфер** — массив байтов `uint8_t*`. Логическая длина `size()` — это число **битов** (сколько значений `bool` хранится), а `capacity()` в этой реализации задаётся в тех же единицах: сколько битов помещается в выделенном буфере без перевыделения.

2. **Индексация по битам** — для элемента с индексом `i` выбирается байт `i / 8` и номер бита `i % 8` внутри байта (младший бит соответствует «первому» биту в этом байте для данного индекса).

3. **`operator[]` для неконстантного вектора** не может вернуть ссылку на отдельный бит (у бита нет адреса в смысле C++). Поэтому возвращается вложенный прокси-класс **`Reference`**: он помнит указатель на байт и номер бита; при чтении выполняется сдвиг и маска, при записи — установка или сброс соответствующего бита.

4. **Константный `operator[]`** возвращает обычный `bool`: значение считывается из упакованного буфера без прокси.

5. **`push_back` / `pop_back`** работают с логическим размером в битах; при нехватке места буфер перевыделяется, размер в битах увеличивается (как правило, с запасом), старые биты копируются в новый массив байтов.

В результате до восьми логических значений `bool` могут делить один байт `uint8_t`, что и требуется от битовой специализации.
